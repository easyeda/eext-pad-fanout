与AI协作开发扩展的使用指南
1. 开始开发前
审阅文档 → 确认方案 → 通知我开始实现
审阅 docs/ 目录下的所有文档，重点确认：
- DOC-002 功能规格（是否符合你的预期）
- DOC-003 技术规格（API使用是否正确）
- DOC-005 代码模板（模板结构是否合适）
2. 日常开发交互
告诉我的格式：
[任务] + [文档引用] + [具体要求]
示例：
"按照 DOC-005 的模板实现 src/index.ts，只实现 Phase 1-2 的功能"
"按照 DOC-002-F003 的要求实现方向计算逻辑"
"发现 DRC 检查不生效，参考 DOC-007-Q13 排查"
常用指令：
你的指令
"开始实现 Phase 1"
"实现核心逻辑"
"实现 UI"
"运行 lint"
"打包测试"
"按照文档测试"

5. 推荐的交互方式
方式一：按Phase实现
"按照 Phase 顺序实现，先完成 Phase 1-2"
"Phase 1 完成，测试通过后继续 Phase 3"
方式二：按模块实现
"先实现 extension.json 配置"
"再实现 src/index.ts 核心逻辑"
"最后实现 iframe/index.html UI"
方式三：按功能实现
"实现菜单注册功能"
"实现焊盘选中逻辑"
"实现实时预览"
"实现 DRC 检查"

git工作流程
你的指令
"功能满意，打个Tag"
"列出所有Tag"
"回退到这个Tag"
"删除Tag"
示例
你：Phase 1-2 完成了，功能满意，打个Tag叫 v0.1
我：git tag v0.1
你：继续优化 Phase 3
我：实现代码
你：出问题了，回退到 v0.1
我：git checkout v0.1 或 git reset --hard v0.1
你：新版本满意了，打 v0.2
我：git tag v0.2
